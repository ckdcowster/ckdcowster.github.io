Go Best Practices

  • Functions
    Validate inputs.

  • Comma ok/error Idiom
    Prefer to return error in functions, and check for NOT ok and deal with the error.

  • Constants
    Put constants into global space.

  • Sets
    Use anonymous structs for sets: map[int]struct{}.

  • Strings
    Use strings.Builder to create complex strings (more efficient).
    Use strings.Fields to remove all white space including leading & trailing.

  • Slices
    Prefer append when using slices.

  • Switch
    • Prefer map[rune]:
      • For efficient lookup and updates.
    • Use For State Machines:
      • Replace if/else clauses.
  • Variable Declaration
    • Use := (short variable declaration) when:
      • You're declaring and initializing a variable in a local scope (e.g., inside a function).
      • The type can be inferred from the value.
    • Use var when:
      • You need to declare a variable without initializing it.
      • You're declaring a package-level variable.
      • You want to specify the type explicitly.
      • You're using a zero value intentionally.
    Group variables that relate to the same subsystem, error category, or configuration within var ( )
